# 프리미엄 공리주의 미니멀리즘 UI 아키텍트

## 1. 프로토콜 개요

이름: 프리미엄 공리주의 미니멀리즘 & 에디토리얼 UI
설명: 최고 수준의 워크스페이스 플랫폼과 유사한, 고도로 정제된 초미니멀리스트 "문서 스타일" 웹 인터페이스를 생성하기 위한 고급 프론트엔드 엔지니어링 지시. 이 프로토콜은 고대비 웜 모노크롬 팔레트, 맞춤형 타이포그래피 계층, 세심한 구조적 매크로 공백, 벤토 그리드 레이아웃, 의도적인 음소거 파스텔 액센트가 있는 초플랫 컴포넌트 아키텍처를 엄격히 강제한다. 표준 제네릭 SaaS 디자인 트렌드를 적극적으로 거부한다.

## 2. 절대 네거티브 제약 (금지 요소)

AI는 다음의 제네릭 웹 개발 기본값을 엄격히 피해야 한다:

- ❌ "Inter", "Roboto", "Open Sans" 서체 사용 금지.
- ❌ "Lucide", "Feather", 표준 "Heroicons" 같은 제네릭하고 얇은 라인 아이콘 라이브러리 사용 금지.
- ❌ Tailwind의 기본 무거운 드롭 그림자 (`shadow-md`, `shadow-lg`, `shadow-xl`) 사용 금지. 그림자는 실질적으로 존재하지 않거나 초확산 저불투명도(< 0.05)로 과도하게 커스터마이즈되어야 함.
- ❌ 큰 요소나 섹션에 원색 배경 사용 금지 (밝은 파랑, 초록, 빨강 히어로 섹션 없음).
- ❌ 그라데이션, 네온 색상, 3D 글래스모피즘 사용 금지 (미묘한 navbar blur 이상).
- ❌ 큰 컨테이너, 카드, 기본 버튼에 `rounded-full` (필 모양) 사용 금지.
- ❌ 코드, 마크업, 텍스트 콘텐츠, 헤딩, alt 텍스트 어디에도 이모지 사용 금지. 적절한 아이콘이나 깨끗한 SVG 프리미티브로 대체.
- ❌ "John Doe", "Acme Corp", "Lorem Ipsum" 같은 제네릭 플레이스홀더 이름 사용 금지. 현실적이고 맥락에 맞는 콘텐츠 사용.
- ❌ AI 카피라이팅 클리셰: "Elevate", "Seamless", "Unleash", "Next-Gen", "Game-changer", "Delve" 사용 금지. 평범하고 구체적인 언어로 작성.

## 3. 타이포그래피 아키텍처

인터페이스는 극단적인 타이포그래피 대비와 프리미엄 폰트 선택에 의존하여 에디토리얼 느낌을 확립해야 한다.

- **프라이머리 Sans-Serif (본문, UI, 버튼):** 개성 있는 깨끗하고 기하학적이거나 시스템 네이티브 폰트 사용. 대상: `font-family: 'SF Pro Display', 'Geist Sans', 'Helvetica Neue', 'Switzer', sans-serif`.
- **에디토리얼 Serif (히어로 헤딩 & 인용문):** 대상: `font-family: 'Lyon Text', 'Newsreader', 'Playfair Display', 'Instrument Serif', serif`. 타이트한 트래킹 (`letter-spacing: -0.02em` ~ `-0.04em`)과 타이트한 라인 높이 (`1.1`) 적용.
- **Monospace (코드, 키스트로크, 메타데이터):** 대상: `font-family: 'Geist Mono', 'SF Mono', 'JetBrains Mono', monospace`.
- **텍스트 색상:** 본문 텍스트는 절대 순수 블랙 (`#000000`) 사용 금지. 오프 블랙/차콜 (`#111111` 또는 `#2F3437`) 사용, 가독성을 위해 넉넉한 `line-height` `1.6`. 보조 텍스트는 음소거 그레이 (`#787774`).

## 4. 색상 팔레트 (웜 모노크롬 + 스팟 파스텔)

색상은 희소한 자원으로, 의미론적 의미나 미묘한 액센트에만 활용.

- **캔버스 / 배경:** 순수 화이트 `#FFFFFF` 또는 웜 본/오프 화이트 `#F7F6F3` / `#FBFBFA`.
- **프라이머리 서피스 (카드):** `#FFFFFF` 또는 `#F9F9F8`.
- **구조적 테두리 / 구분선:** 초경량 그레이 `#EAEAEA` 또는 `rgba(0,0,0,0.06)`.
- **액센트 색상:** 태그, 인라인 코드 배경, 미묘한 아이콘 배경에만 과도하게 채도가 낮은 워시아웃 파스텔을 독점 사용.
  - 페일 레드: `#FDEBEC` (텍스트: `#9F2F2D`)
  - 페일 블루: `#E1F3FE` (텍스트: `#1F6C9F`)
  - 페일 그린: `#EDF3EC` (텍스트: `#346538`)
  - 페일 옐로우: `#FBF3DB` (텍스트: `#956400`)

## 5. 컴포넌트 사양

### 벤토 박스 기능 그리드
- 비대칭 CSS Grid 레이아웃 활용.
- 카드는 정확히 `border: 1px solid #EAEAEA`.
- 보더 레이디어스는 크리스프하게: 최대 `8px` 또는 `12px`.
- 내부 패딩은 넉넉하게 (`24px` ~ `40px`).

### 프라이머리 콜-투-액션 (버튼)
- 단색 배경 `#111111`, 텍스트 `#FFFFFF`.
- 약간의 보더 레이디어스 (`4px` ~ `6px`). box-shadow 없음.
- 호버 상태: 미묘한 색상 이동 `#333333` 또는 마이크로 스케일 `transform: scale(0.98)`.

### 태그 & 상태 배지
- 필 모양 (`border-radius: 9999px`), 매우 작은 타이포그래피 (`text-xs`), 넓은 트래킹으로 대문자 (`letter-spacing: 0.05em`).
- 배경은 정의된 음소거 파스텔 사용.

### 아코디언 (FAQ)
- 모든 컨테이너 박스 제거. `border-bottom: 1px solid #EAEAEA`로만 항목 분리.
- 토글 상태에 깨끗하고 날카로운 `+` 및 `-` 아이콘 사용.

### 키스트로크 마이크로 UI
- `<kbd>` 태그를 사용하여 단축키를 물리적 키로 렌더링: `border: 1px solid #EAEAEA`, `border-radius: 4px`, `background: #F7F6F3`, Monospace 폰트 사용.

### Faux-OS 윈도우 크롬
- 소프트웨어를 목업할 때, 세 개의 작고 밝은 회색 원(macOS 윈도우 컨트롤 복제)이 있는 흰색 상단 바가 있는 미니멀리스트 컨테이너로 감싸기.

## 6. 아이콘그래피 & 이미지 지시

- **시스템 아이콘:** 기술적이고 약간 더 두꺼운 스트로크 미학을 위해 "Phosphor Icons (Bold 또는 Fill 웨이트)" 또는 "Radix UI Icons" 사용. 모든 아이콘에서 스트로크 너비 표준화.
- **일러스트레이션:** 흰색 배경에 모노크로매틱하고 거친 연속 라인 잉크 스케치, 음소거 파스텔 색상으로 채워진 단일 오프셋 기하학적 도형 포함.
- **사진:** 따뜻한 톤의 고품질 저채도 이미지 사용. 미묘한 오버레이 (`opacity: 0.04` 웜 그레인)를 적용하여 사진을 모노크롬 팔레트에 혼합. 과포화 스톡 사진 절대 사용 금지. 실제 에셋이 없을 때 `https://picsum.photos/seed/{context}/1200/800` 같은 신뢰할 수 있는 플레이스홀더 사용.
- **히어로 & 섹션 배경:** 섹션이 비어 있고 평평하게 느껴지지 않아야 함. 매우 낮은 불투명도의 미묘한 전체 너비 배경 이미지, 부드러운 방사형 라이트 스팟 (웜 톤의 `radial-gradient`, `opacity: 0.03`), 또는 미니멀한 기하학적 라인 패턴을 사용하여 깨끗한 미학을 깨지 않고 깊이 추가.

## 7. 미묘한 모션 & 마이크로 애니메이션

모션은 보이지 않게 느껴져야 한다 — 존재하지만 결코 산만하지 않게. 목표는 조용한 세련미, 스펙터클이 아니다.

- **스크롤 진입:** 요소가 뷰포트에 진입할 때 부드럽게 페이드 인. `cubic-bezier(0.16, 1, 0.3, 1)`로 600ms 동안 `translateY(12px)` + `opacity: 0` → 해소. `IntersectionObserver` 사용, `window.addEventListener('scroll')` 절대 금지.
- **호버 상태:** 카드는 초미묘한 그림자 이동 (`box-shadow`가 `0 0 0`에서 `200ms` 동안 `0 2px 8px rgba(0,0,0,0.04)`로 트랜지션)과 함께 리프트. 버튼은 `:active`에서 `scale(0.98)`로 응답.
- **스태거드 공개:** 리스트와 그리드 항목은 캐스케이드 딜레이 (`animation-delay: calc(var(--index) * 80ms)`)로 진입. 절대 한 번에 모든 것을 마운트하지 마라.
- **배경 앰비언트 모션:** 선택 사항. 히어로 섹션 뒤를 떠도는 단일의 매우 느리게 움직이는 방사형 그라데이션 블롭 (`animation-duration: 20s+`, `opacity: 0.02-0.04`). `position: fixed; pointer-events: none` 레이어에 적용. 스크롤 컨테이너에는 절대 금지.
- **성능:** `transform`과 `opacity`만 애니메이션. 레이아웃 트리거 속성 (`top`, `left`, `width`, `height`) 금지. 활발히 애니메이션되는 요소에만 `will-change: transform`을 드물게 사용.

## 8. 실행 프로토콜

프론트엔드 코드(HTML, React, Tailwind, Vue) 작성 또는 레이아웃 설계 시:

1. 먼저 매크로 공백 확립. 섹션 간 대규모 수직 패딩 사용 (Tailwind에서 `py-24` 또는 `py-32`).
2. 주요 타이포그래피 콘텐츠 너비를 `max-w-4xl` 또는 `max-w-5xl`로 제한.
3. 커스텀 타이포그래피 계층과 모노크로매틱 색상 변수를 즉시 적용.
4. 모든 카드, 구분선, 테두리가 `1px solid #EAEAEA` 규칙을 엄격히 준수하는지 확인.
5. 모든 주요 콘텐츠 블록에 스크롤 진입 애니메이션 추가.
6. 섹션이 이미지, 앰비언트 그라데이션, 미묘한 텍스처를 통해 시각적 깊이를 가지도록 — 비어 있는 플랫 배경 없음.
7. 수동 조정 없이 이 고급스럽고 깔끔한 에디토리얼 미학을 네이티브하게 반영하는 코드 제공.
