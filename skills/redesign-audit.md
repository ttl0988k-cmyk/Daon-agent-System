# 리디자인 스킬

## 작동 방식

기존 프로젝트에 적용할 때 이 순서를 따른다:

1. **스캔** — 코드베이스를 읽어라. 프레임워크, 스타일링 방법(Tailwind, vanilla CSS, styled-components 등), 현재 디자인 패턴을 식별하라.
2. **진단** — 아래 감사를 실행하라. 발견한 모든 제네릭 패턴, 약점, 누락된 상태를 나열하라.
3. **수정** — 기존 스택과 함께 작동하는 대상화된 업그레이드를 적용하라. 처음부터 다시 작성하지 마라. 있는 것을 개선하라.

## 디자인 감사

### 타이포그래피

다음 문제를 확인하고 수정하라:

- **브라우저 기본 폰트 또는 사방에 Inter.** 개성 있는 폰트로 교체. 좋은 옵션: `Geist`, `Outfit`, `Cabinet Grotesk`, `Satoshi`. 에디토리얼/크리에이티브 프로젝트는 세리프 헤더와 산세리프 본문 페어링.
- **헤드라인에 존재감 부족.** 디스플레이 텍스트 크기 증가, letter-spacing 타이트, line-height 감소. 헤드라인은 무겁고 의도적으로 느껴져야 함.
- **본문 텍스트 너무 넓음.** 단락 너비를 약 65자로 제한. 가독성을 위해 line-height 증가.
- **Regular(400)와 Bold(700) 웨이트만 사용.** Medium(500)과 SemiBold(600) 도입으로 더 미묘한 계층 구조.
- **숫자가 프로포셔널 폰트.** 데이터가 많은 인터페이스에서 모노스페이스 폰트 사용 또는 표 형식 숫자 활성화 (`font-variant-numeric: tabular-nums`).
- **letter-spacing 조정 누락.** 대형 헤더에 네거티브 트래킹, 작은 캡스나 라벨에 포지티브 트래킹.
- **모든 곳에 올캡스 서브헤더.** 대신 소문자 이탤릭, 문장 케이스, 또는 스몰 캡스 시도.
- **고아 단어.** 마지막 줄에 혼자 있는 단일 단어. `text-wrap: balance` 또는 `text-wrap: pretty`로 수정.

### 색상 및 표면

- **순수 `#000000` 배경.** 오프 블랙, 다크 차콜, 또는 틴트 다크 (`#0a0a0a`, `#121212`, 또는 다크 네이비)로 교체.
- **과포화 액센트 색상.** 채도를 80% 이하로 유지. 액센트가 중립과 조화되도록 채도 낮추기.
- **하나 이상의 액센트 색상.** 하나를 선택. 나머지는 제거. 일관성이 다양성을 이긴다.
- **웜과 쿨 그레이 혼합.** 하나의 그레이 계열만 사용. 모든 그레이를 일관된 색조(웜 또는 쿨, 둘 다 아님)로 틴트.
- **퍼플/블루 "AI 그라데이션" 미학.** 이것은 가장 흔한 AI 디자인 지문. 중립 베이스와 단일의 고려된 액센트로 교체.
- **제네릭 `box-shadow`.** 배경 색조에 맞게 그림자 착색. 순수 블랙 저불투명도 대신 착색된 그림자 사용 (예: 파란 배경에 다크 블루 그림자).
- **질감 없는 플랫 디자인.** 배경에 미묘한 노이즈, 그레인, 마이크로 패턴 추가. 순수 플랫 벡터는 무균적으로 느껴짐.
- **완벽하게 균일한 그라데이션.** 표준 선형 45도 페이드 대신 방사형 그라데이션, 노이즈 오버레이, 메쉬 그라데이션으로 균일성 깨기.
- **일관성 없는 조명 방향.** 모든 그림자가 단일의 일관된 광원을 암시하는지 감사.
- **라이트 모드 페이지의 무작위 다크 섹션 (또는 그 반대).** 그렇지 않으면 라이트 페이지를 깨는 단일 다크 배경 섹션은 복사-붙여넣기 사고처럼 보임. 전체 다크 모드에 전념하거나 전체적으로 일관된 배경 톤 유지. 대비가 필요하면 동일한 팔레트의 약간 더 어두운 음영 사용 — 크림 페이지 중간에 갑자기 `#111`로 점프하지 마라.
- **시각적 깊이 없는 비어 있고 플랫한 섹션.** 플레인 배경에 텍스트만 있는 섹션은 미완성처럼 느껴짐. 고품질 배경 이미지 (블러, 오버레이, 마스크), 미묘한 패턴, 앰비언트 그라데이션 추가. 실제 에셋이 없을 때 `https://picsum.photos/seed/{name}/1920/1080` 같은 신뢰할 수 있는 플레이스홀더 소스 사용.

### 레이아웃

- **모든 것이 중앙 정렬되고 대칭적.** 오프셋 마진, 혼합 종횡비, 중앙 콘텐츠 위의 왼쪽 정렬 헤더로 대칭 깨기.
- **기능 행으로 세 개의 동일한 카드 열.** 이것은 가장 제네릭한 AI 레이아웃. 2열 지그재그, 비대칭 그리드, 가로 스크롤, 메이슨리 레이아웃으로 교체.
- **전체 화면 섹션에 `height: 100vh` 사용.** 모바일 브라우저 레이아웃 점프 방지를 위해 `min-height: 100dvh`로 교체 (iOS Safari 뷰포트 버그).
- **복잡한 flexbox 백분율 계산.** 신뢰할 수 있는 다중 열 구조를 위해 CSS Grid로 교체.
- **최대 너비 컨테이너 없음.** 콘텐츠가 넓은 화면에서 가장자리까지 늘어나지 않도록 컨테이너 제약 (약 1200-1440px)과 자동 마진 추가.
- **Flexbox로 강제된 동일 높이 카드.** 콘텐츠 길이가 다를 때 가변 높이 허용 또는 메이슨리 사용.
- **모든 것에 균일한 border-radius.** 반경 다양화: 내부 요소는 더 타이트하게, 컨테이너는 더 부드럽게.
- **중첩이나 깊이 없음.** 요소가 서로 평평하게 옆에 있음. 네거티브 마진을 사용하여 레이어링과 깊이 생성.

### 아이콘그래피

- **Lucide 또는 Feather 아이콘만 사용.** 이것들은 "기본" AI 아이콘 선택. 차별화를 위해 Phosphor, Heroicons, 커스텀 세트 사용.
- **"출시"에 로켓, "보안"에 방패.** 덜 뻔한 아이콘으로 클리셰 메타포 교체 (볼트, 지문, 스파크, 볼트).
- **일관성 없는 스트로크 너비.** 모든 아이콘 감사하고 하나의 스트로크 웨이트로 표준화.
- **파비콘 누락.** 항상 브랜드 파비콘 포함.
- **스톡 "다양한 팀" 사진.** 불쾌한 스톡 이미지 대신 실제 팀 사진, 캔디드 샷, 일관된 일러스트레이션 스타일 사용.

### 코드 품질

- **Div 수프.** 시맨틱 HTML 사용: `<nav>`, `<main>`, `<article>`, `<aside>`, `<section>`.
- **CSS 클래스와 혼합된 인라인 스타일.** 모든 스타일링을 프로젝트의 스타일링 시스템으로 이동.
- **하드코딩된 픽셀 너비.** 유연한 레이아웃을 위해 상대 단위 (`%`, `rem`, `em`, `max-width`) 사용.
- **이미지에 alt 텍스트 누락.** 스크린 리더를 위해 이미지 콘텐츠 설명. 의미 있는 이미지에 `alt=""` 또는 `alt="image"` 남기지 마라.
- **`9999` 같은 임의의 z-index 값.** 테마/변수에 깨끗한 z-index 스케일 설정.
- **주석 처리된 죽은 코드.** 배포 전 모든 디버그 아티팩트 제거.
- **임포트 환각.** 모든 임포트가 실제로 `package.json` 또는 프로젝트 의존성에 존재하는지 확인.
- **메타 태그 누락.** 적절한 `<title>`, `description`, `og:image`, 소셜 공유 메타 태그 추가.

### 전략적 누락 (AI가 일반적으로 잊는 것)

- **법적 링크 없음.** 푸터에 개인정보 처리방침 및 이용약관 링크 추가.
- **"뒤로" 탐색 없음.** 사용자 흐름의 막다른 길. 모든 페이지는 돌아갈 방법이 필요.
- **커스텀 404 페이지 없음.** 도움이 되고 브랜드화된 "페이지를 찾을 수 없음" 경험 설계.
- **폼 유효성 검사 없음.** 이메일, 필수 필드, 형식 확인을 위한 클라이언트 측 유효성 검사 추가.
- **"콘텐츠로 건너뛰기" 링크 없음.** 키보드 사용자에게 필수. 숨겨진 skip-link 추가.
- **쿠키 동의 없음.** 관할권에서 요구하는 경우 규정 준수 동의 배너 추가.

## 업그레이드 기법

프로젝트 업그레이드 시, 제네릭 패턴을 교체하기 위해 이 고임팩트 기법들에서 가져와라:

### 타이포그래피 업그레이드
- **가변 폰트 애니메이션.** 스크롤이나 호버에서 웨이트 또는 너비 보간 — 살아있는 듯한 텍스트.
- **아웃라인에서 채우기로 트랜지션.** 텍스트가 스트로크 아웃라인으로 시작하여 스크롤 진입 또는 인터랙션에서 색상으로 채워짐.
- **텍스트 마스크 공개.** 뒤의 비디오나 애니메이션 이미지에 대한 창 역할을 하는 대형 타이포그래피.

### 레이아웃 업그레이드
- **깨진 그리드 / 비대칭.** 의도적으로 열 구조를 무시하는 요소 — 중첩, 화면 밖으로 블리드, 계산된 무작위성으로 오프셋.
- **공백 극대화.** 단일 요소에 집중을 강제하는 공격적인 네거티브 스페이스 사용.
- **패럴랙스 카드 스택.** 스크롤 중에 서로 물리적으로 붙고 쌓이는 섹션.
- **분할 화면 스크롤.** 반대 방향으로 슬라이딩하는 화면의 두 반쪽.

### 모션 업그레이드
- **관성 있는 부드러운 스크롤.** 더 무겁고 시네마틱한 느낌을 위해 브라우저 기본값에서 스크롤 분리.
- **스태거드 진입.** 요소가 약간의 딜레이로 캐스케이드, Y축 트랜슬레이션과 불투명도 페이드 결합. 절대 한 번에 모든 것을 마운트하지 마라.
- **스프링 물리.** 모든 인터랙티브 요소에서 자연스럽고 무게감 있는 느낌을 위해 선형 이징을 스프링 기반 모션으로 교체.
- **스크롤 구동 공개.** 스크롤 진행에 연결된 확장 마스크, 와이프, SVG 경로 그리기를 통해 진입하는 콘텐츠.

### 표면 업그레이드
- **진정한 글래스모피즘.** `backdrop-filter: blur`를 넘어서. 가장자리 굴절을 시뮬레이션하기 위한 1px 내부 테두리와 미묘한 내부 그림자 추가.
- **스포트라이트 테두리.** 커서 아래에서 동적으로 조명되는 카드 테두리.
- **그레인 및 노이즈 오버레이.** 디지털 플랫함을 깨기 위한 미묘한 노이즈가 있는 고정된 pointer-events-none 오버레이.
- **착색된 틴트 그림자.** 제네릭 블랙 대신 배경의 색조를 담은 그림자.

## 수정 우선순위

최대 시각적 임팩트와 최소 위험을 위해 이 순서로 변경 적용:

1. **폰트 교체** — 가장 큰 즉각적 개선, 가장 낮은 위험
2. **색상 팔레트 정리** — 충돌하거나 과포화된 색상 제거
3. **호버 및 활성 상태** — 인터페이스가 살아있는 느낌
4. **레이아웃 및 간격** — 적절한 그리드, 최대 너비, 일관된 패딩
5. **제네릭 컴포넌트 교체** — 클리셰 패턴을 현대적 대안으로 교체
6. **로딩, 빈 상태, 오류 상태 추가** — 완성된 느낌
7. **타이포그래피 스케일 및 간격 폴리시** — 프리미엄 최종 터치

## 규칙

- 기존 기술 스택으로 작업할 것. 프레임워크나 스타일링 라이브러리를 마이그레이션하지 마라.
- 기존 기능을 깨지 마라. 모든 변경 후 테스트하라.
- 새 라이브러리를 임포트하기 전에 프로젝트 의존성 파일을 먼저 확인하라.
