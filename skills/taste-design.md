# Taste-Skill: Anti-Slop Frontend Design

> 랜딩페이지, 포트폴리오, 리디자인 대상. 대시보드, 데이터 테이블, 멀티스텝 제품 UI에는 적용하지 않는다.
> 모든 규칙은 **맥락적(contextual)** 이다. 자동으로 발동하지 않는다. 먼저 브리핑을 읽고, 필요한 것만 발췌하라.

---

## 0. 브리핑 추론 (작업 전 맥락 파악)

코드를 작성하기 전에, **사용자가 실제로 원하는 것이 무엇인지 추론하라**. 대부분의 LLM 디자인 출력이 형편없는 이유는 모델이 기본 미학으로 바로 점프하기 때문이다.

### 0.A 이 신호들을 먼저 읽어라
1. **페이지 종류** - 랜딩 (SaaS / 컨슈머 / 에이전시 / 이벤트), 포트폴리오 (개발자 / 디자이너 / 크리에이티브 스튜디오), 리디자인 (보존 vs 전면개편), 에디토리얼 / 블로그.
2. **사용자가 사용한 분위기 단어** - "미니멀리스트", "차분한", "Linear 스타일", "Awwwards", "브루탈리스트", "프리미엄 컨슈머", "Apple 스러운", "플레이풀", "진지한 B2B", "에디토리얼", "에이전시 느낌", "글래시", "다크 테크".
3. **레퍼런스 신호** - 링크된 URL, 붙여넣은 스크린샷, 언급된 제품, 경쟁 브랜드.
4. **대상** - B2B 구매 담당자 vs. 디자인 감각 있는 소비자 vs. 포트폴리오를 훑는 채용 담당자. 대상이 미학을 결정한다.
5. **이미 존재하는 브랜드 에셋** - 로고, 색상, 타이포그래피, 사진. 리디자인 시 이것들은 선택적 입력이 아닌 시작 재료다.
6. **조용한 제약** - 접근성 최우선 대상, 공공 부문, 규제 산업, 신뢰 우선 커머스, 아동 제품. 이 제약들은 미학적 선호를 **오버라이드**한다.

### 0.B 코드 생성 전 "디자인 리드" 한 줄 선언
코드 작성 전, 한 줄로 선언하라: **"이것은: \<대상>을 위한 \<페이지 종류>, \<분위기> 언어, \<디자인 시스템 또는 미학 계열>에 기대어 읽는다."**

예시:
- *"이것은: 기술 구매자를 위한 B2B SaaS 랜딩, Linear 스타일 미니멀리스트 언어, Tailwind 유틸리티 + Geist + 절제된 모션에 기대어 읽는다."*
- *"이것은: 채용 담당자를 위한 솔로 디자이너 포트폴리오, 에디토리얼/키네틱 타입 언어, 네이티브 CSS + 스크롤 기반 애니메이션 + 커스텀 타이포그래피에 기대어 읽는다."*
- *"이것은: 공공 부문 서비스 사이트 리디자인, 신뢰 우선 언어, GOV.UK Frontend 또는 USWDS에 기대어 읽는다."*

### 0.C 브리핑이 모호하면 하나의 질문만, 추측하지 마라
정확히 **하나**의 명확한 질문 - 절대 여러 질문을 한 번에 던지지 마라 - 그리고 디자인 리드가 진정으로 갈릴 때만 질문하라. 예: *"Linear-클린에 가까워야 할까요, 아니면 Awwwards-실험적에 가까워야 할까요?"*

맥락에서 자신 있게 추론할 수 있다면, **묻지 마라**. 그냥 디자인 리드를 선언하고 진행하라.

### 0.D 기본값 방지 훈련
AI-퍼플 그라데이션, 다크 메쉬 위 중앙 정렬 히어로, 동일한 세 개의 기능 카드, 모든 것에 제네릭 글래스모피즘, 모든 곳에 무한 루프 마이크로 애니메이션, Inter + slate-900으로 기본 설정하지 마라. 이것들은 LLM 기본값이다. 디자인 리드에 기반해 의도적으로 그것들을 넘어서라.

## 1. 세 가지 다이얼 (핵심 설정)

디자인 리드 후, 세 가지 다이얼을 설정하라. 아래의 모든 레이아웃, 모션, 밀도 결정은 이 다이얼에 의해 제어된다.

* **`DESIGN_VARIANCE: 8`** - 1 = 완벽한 대칭, 10 = 예술적 혼돈
* **`MOTION_INTENSITY: 6`** - 1 = 정적, 10 = 시네마틱 / 물리 기반
* **`VISUAL_DENSITY: 4`** - 1 = 아트 갤러리 / 통풍, 10 = 콕핏 / 데이터 밀집

**기본값:** `8 / 6 / 4`. 디자인 리드가 오버라이드하지 않는 한 이것들을 사용하라.

### 1.A 다이얼 추론 (디자인 리드 → 다이얼 값)
| 신호 | VARIANCE | MOTION | DENSITY |
|---|---|---|---|
| "미니멀리스트 / 클린 / 차분한 / 에디토리얼 / Linear 스타일" | 5-6 | 3-4 | 2-3 |
| "프리미엄 컨슈머 / Apple 스러운 / 럭셔리 / 브랜드" | 7-8 | 5-7 | 3-4 |
| "플레이풀 / 와일드 / Dribbble / Awwwards / 실험적 / 에이전시" | 9-10 | 8-10 | 3-4 |
| "랜딩페이지 / 포트폴리오 / 마케팅 사이트 (기본값)" | 7-9 | 6-8 | 3-5 |
| "신뢰 우선 / 공공 부문 / 규제 / 접근성 중요" | 3-4 | 2-3 | 4-5 |
| "리디자인 - 보존" | 기존과 동일 | +1 | 기존과 동일 |
| "리디자인 - 전면개편" | +2 | +2 | 기존과 동일 |

### 1.B 유즈케이스 프리셋
| 유즈케이스 | VARIANCE | MOTION | DENSITY |
|---|---|---|---|
| 랜딩 (SaaS, 메인스트림) | 7 | 6 | 4 |
| 랜딩 (에이전시 / 크리에이티브) | 9 | 8 | 3 |
| 랜딩 (프리미엄 컨슈머) | 7 | 6 | 3 |
| 포트폴리오 (디자이너 / 스튜디오) | 8 | 7 | 3 |
| 포트폴리오 (개발자) | 6 | 5 | 4 |
| 에디토리얼 / 블로그 | 6 | 4 | 3 |
| 공공 부문 서비스 | 3 | 2 | 5 |
| 리디자인 - 보존 | 일치 | 일치+1 | 일치 |
| 리디자인 - 전면개편 | +2 | +2 | 일치 |

### 1.C 다이얼이 출력을 구동하는 방식
이 값들(또는 사용자가 오버라이드한 값)을 전역 변수로 사용하라. 이 문서 전반의 상호 참조는 이 정확한 변수명을 참조한다.

## 2. 타이포그래피 아키텍처

### 필수 규칙
- **디스플레이/헤드라인:** 트래킹 타이트, 제어된 스케일. 무게와 색상을 통한 계층 구조.
- **본문:** 여유로운 리딩, 최대 65자 너비.
- **Inter 금지:** 프리미엄/크리에이티브 맥락에서는 Inter를 사용하지 마라. `Geist`, `Outfit`, `Cabinet Grotesk`, `Satoshi` 사용.
- **일반 세리프 금지:** `Times New Roman`, `Georgia`, `Garamond`, `Palatino` 사용 금지. 세리프가 필요하면 `Fraunces`, `Gambarino`, `Editorial New`, `Instrument Serif`만 사용.
- **대시보드 제약:** Sans-Serif 페어링만 사용 (`Geist` + `Geist Mono` 또는 `Satoshi` + `JetBrains Mono`).
- **고밀도 오버라이드:** DENSITY가 7을 초과하면 모든 숫자는 Monospace.

---

## 3. 색상 시스템

### 절대 금지
- **AI 퍼플/블루 네온 미학** — 보라색 버튼 글로우, 네온 그라데이션 없음
- **순수 블랙 (`#000000`)** — Off-Black, Zinc-950, 또는 Charcoal 사용
- **과포화 액센트** — 채도 80% 이하로 유지
- **하나 이상의 액센트 색상** — 하나를 고르고 나머지는 제거

### 원칙
- 절대적인 중립 베이스 (Zinc/Slate) + 고대비 단일 액센트
- 하나의 팔레트를 전체 출력에 고수 — 웜/쿨 그레이 혼합 금지
- 그림자는 배경 색조에 맞춰 착색 — 순수 블랙 낮은 불투명도 대신
- 플랫 디자인에 질감 추가 — 미묘한 노이즈, 그레인, 마이크로 패턴

---

## 4. 레이아웃 원칙

### 금지 패턴
- **3개의 동일한 카드 열** — 가장 제네릭한 AI 레이아웃. 2열 지그재그, 비대칭 그리드, 가로 스크롤, 또는 메이슨리로 대체.
- **중앙 정렬 히어로** — VARIANCE가 4를 초과하면 중앙 정렬 히어로 금지.
- **`height: 100vh`** — `min-height: 100dvh`로 대체 (iOS Safari 뷰포트 버그).
- **복잡한 flexbox 백분율 계산** — CSS Grid로 대체.

### 원칙
- CSS Grid 아키텍처 우선
- 그리드 트랙과 교차점에 요소 정확히 배치
- 극단적인 데이터 밀도와 계산된 네거티브 스페이스 사이를 오가기
- `border-radius` 일관성 버리기 — 내부 요소는 더 타이트하게, 컨테이너는 더 부드럽게
- 섹션 패딩 최소 `py-24` — 레이아웃이 숨 쉬게 하라

---

## 5. 모션 철학

- **스프링 물리 기본값:** `cubic-bezier(0.32, 0.72, 0, 1)` — 프리미엄, 무게감 있는 느낌. 선형 이징 없음.
- **스태거드 오케스트레이션:** 리스트를 한 번에 마운트하지 마라 — 워터폴 공개를 위해 캐스케이드 딜레이 사용.
- **성능:** `transform`과 `opacity`만 애니메이션. `top`, `left`, `width`, `height` 절대 애니메이션 금지.
- **블러 제약:** `backdrop-blur`는 고정/스티키 요소에만 적용. 스크롤 컨테이너에는 절대 금지.
- **그레인/노이즈 오버레이:** 고정된 `pointer-events-none` 가상 요소에만.

---

## 6. 컴포넌트 표준

### 버튼
- 플랫, 외부 글로우 없음
- active 상태에서 촉각적 -1px translate
- 프라이머리는 액센트 채우기, 세컨더리는 고스트/아웃라인

### 카드
- 넉넉한 라운드 코너 (2.5rem)
- 확산된 위스퍼 그림자
- 계층 구조가 필요할 때만 사용
- 고밀도: border-top 구분선으로 대체

### 입력/폼
- 입력 위에 레이블, 아래에 오류
- 포커스 링은 액센트 색상
- 플로팅 레이블 없음

### 로더
- 레이아웃 차원과 정확히 일치하는 스켈레톤 쉬머
- 원형 스피너 없음

## 7. 안티 패턴 (AI 흔적)

이것들은 **절대** 하지 마라:
- ❌ 이모지 사용
- ❌ Inter 폰트
- ❌ 일반 세리프 폰트 (Times New Roman, Georgia, Garamond)
- ❌ 순수 블랙 (`#000000`)
- ❌ 네온/외부 글로우 그림자
- ❌ 과포화 액센트
- ❌ 대형 헤더의 과도한 그라데이션 텍스트
- ❌ 커스텀 마우스 커서
- ❌ 요소 중첩 — 항상 깨끗한 공간 분리
- ❌ 3열 동일 카드 레이아웃
- ❌ 제네릭 이름 ("John Doe", "Acme", "Nexus")
- ❌ 가짜 라운드 숫자 (`99.99%`, `50%`)
- ❌ AI 카피 클리셰 ("Elevate", "Seamless", "Unleash", "Next-Gen")
- ❌ 필러 UI 텍스트: "Scroll to explore", "Swipe down", 스크롤 화살표
- ❌ 깨진 Unsplash 링크 — `picsum.photos` 또는 SVG 아바타 사용
- ❌ 중앙 정렬 히어로 섹션 (고 VARIANCE 프로젝트에서)

## 8. 실행 프로토콜

UI 코드 생성 시 이 정확한 순서를 따르라:

1. **[묵시적 사고]** 디자인 리드를 추론하고 다이얼을 설정하라.
2. **[스캐폴드]** 배경 텍스처, 매크로 공백 스케일, 대규모 타이포그래피 크기 설정.
3. **[아키텍처]** 엄격한 그리드 아키텍처 구축. 모든 주요 카드/컨테이너는 중첩 구조.
4. **[안무]** 커스텀 큐빅-베지어 트랜지션, 스태거드 공개, 버튼 물리 주입.
5. **[출력]** 완벽한 프론트엔드 코드 제공. 제네릭 폴백 포함하지 마라.

## 9. 사전 출력 체크리스트

코드 제출 전 이 매트릭스로 평가하라:
- [ ] 금지된 폰트, 아이콘, 그림자, 레이아웃, 모션 패턴이 없는가
- [ ] 디자인 리드가 선언되었는가
- [ ] 다이얼(VARIANCE, MOTION, DENSITY)이 설정되었는가
- [ ] 섹션 패딩이 최소 `py-24`인가
- [ ] 모든 트랜지션이 커스텀 큐빅-베지어를 사용하는가
- [ ] 스크롤 진입 애니메이션이 존재하는가
